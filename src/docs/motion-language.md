# Motion Language

Status: 🔒 Frozen

Version: 1.0

This document defines the universal motion philosophy of the project. It is not tied to any specific section or implementation. Instead, it establishes the emotional, visual, and behavioral language that every animation throughout the website must follow.

Motion is never decorative.

Motion exists only to guide attention, reinforce narrative, and communicate atmosphere.

If an animation does not improve understanding, rhythm, or emotion, it should not exist.

---

# 1. Motion Philosophy

Movement should feel inevitable.

Nothing should appear because "an animation starts."

Instead, everything should behave as though it already exists and is slowly revealing itself through time.

Users should rarely notice the animation itself.

They should notice the feeling it creates.

Motion must disappear behind the experience.

---

# 2. Emotional Characteristics

Motion throughout the project should feel:

- Calm
- Editorial
- Organic
- Patient
- Confident
- Quiet
- Intentional
- Natural

Motion must never feel:

- Playful
- Mechanical
- Bouncy
- Aggressive
- Fast
- Flashy
- Hyperactive
- Gamified

---

# 3. Primary Purpose of Motion

Motion exists only to:

- guide attention
- reveal hierarchy
- reinforce narrative
- connect sections
- establish rhythm
- communicate spatial relationships

Motion never exists merely to impress.

---

# 4. Attention Hierarchy

Whenever multiple elements appear together,
attention must follow this order:

Environment

↓

Primary Subject

↓

Secondary Detail

↓

Supporting Text

↓

Interactive Element

Animation should reinforce this hierarchy.

Never compete against it.

---

# 5. Motion Tempo

The overall pacing is slow.

Users should feel encouraged to pause.

Animations should create moments of observation rather than excitement.

The website should never encourage rapid scrolling.

---

# 6. Motion Types

Preferred motion includes:

- fading
- resolving
- uncovering
- growing
- breathing
- drifting
- settling
- diffusion
- erosion
- gentle parallax

These movements resemble natural processes rather than mechanical systems.

---

# 7. Forbidden Motion

The following motion styles are prohibited:

- bounce
- elastic
- overshoot
- spring explosions
- spinning
- flipping
- shaking
- zoom bursts
- exaggerated scaling
- rapid oscillation
- cartoon motion
- surprise animation

Nothing should feel playful.

Nothing should feel artificial.

---

# 8. Spatial Motion

Motion should respect spatial depth.

Foreground moves most.

Midground moves less.

Background moves least.

Parallax should be subtle enough that users feel depth before consciously noticing movement.

Depth should emerge naturally.

---

# 9. Reading Before Motion

Motion must never interrupt reading.

Instead:

Motion creates space

↓

User observes

↓

User reads

↓

Motion quietly disappears

Reading always has priority.

---

# 10. Timing Philosophy

Animations begin gently.

Animations end gently.

There should never be sudden starts.

There should never be abrupt stops.

Acceleration and deceleration should feel continuous.

---

# 11. Section Ownership

Every section owns its own motion.

No section controls another section.

No global storytelling timeline exists.

Motion boundaries follow architectural boundaries.

---

# 12. Timeline Structure

Each section follows:

Introduction

↓

Observation

↓

Focus

↓

Resolution

↓

Transition

This rhythm should remain recognizable throughout the website.

---

# 13. Scroll Philosophy

Scrolling is storytelling.

Animations respond to reading progression rather than viewport mechanics.

ScrollTrigger exists only when it reinforces narrative progression.

Avoid attaching animations merely because an element enters the viewport.

---

# 14. Accessibility

Reduced Motion is a first-class requirement.

When reduced motion is enabled:

- no information disappears
- no interaction breaks
- no layout changes
- no hidden content
- no dependence on animation

Motion is enhancement.

Never functionality.

---

# 15. Performance Philosophy

Only animate:

- transform
- opacity

Never animate layout properties.

Avoid expensive paints.

Prefer fewer meaningful animations over many decorative ones.

---

# 16. Transition Philosophy

Transitions connect emotional states.

They do not simply separate sections.

Every transition should answer:

"What emotional state should the user leave?"

"What emotional state should they enter?"

---

# 17. Consistency

Every section should feel like it belongs to the same film.

Different chapters may have different pacing,

but they must always speak the same motion language.

Users should never feel they have entered another website.

---

# 18. Success Criteria

The motion language succeeds when:

- users remember the atmosphere instead of the animations
- motion naturally guides attention
- reading feels effortless
- scrolling becomes slower
- transitions feel inevitable
- animations disappear behind the story

The highest compliment is not:

"Those animations are amazing."

The highest compliment is:

"This experience feels alive."
