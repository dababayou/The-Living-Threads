# Surface Language

Status: 🔒 Design System

This document defines the visual language of every surface within the project. It exists independently from Story Freeze, Layout Brief, and Motion Language. It describes how objects should appear before they move.

---

# Philosophy

The interface should never resemble a modern software dashboard.

It should feel closer to an editorial publication, museum exhibition, or printed art book.

Surfaces are quiet.

Textures carry meaning.

Materials reveal time.

Nothing should feel synthetic.

---

# Surface Hierarchy

Every visual element belongs to one of four layers.

## Layer 1 — Atmosphere

Purpose:
Create space.

Examples:
- empty background
- gradients
- environmental textures
- light
- fog
- paper grain

Characteristics

- lowest contrast
- largest scale
- almost invisible
- never attracts attention

Atmosphere should be felt before it is noticed.

---

## Layer 2 — Subject

Purpose:
Carry the story.

Examples

- landscape
- fabric
- fiber
- bark
- indigo
- weaving tools
- portraits

Characteristics

- highest visual importance
- calm composition
- generous breathing room
- realistic lighting

Subjects should feel observed, never displayed.

---

## Layer 3 — Annotation

Purpose:
Whisper additional context.

Examples

- captions
- field notes
- labels
- observations

Characteristics

- very small
- subtle borders
- attached directly to imagery
- never floating independently

Annotations should behave like notes written inside a museum archive.

---

## Layer 4 — Structural Guides

Purpose:
Quietly organize the page.

Examples

- thin divider lines
- crosshair intersections
- subtle borders
- invisible alignment systems

Characteristics

- almost disappear
- never decorative
- support composition only

---

# Texture Philosophy

Textures should communicate age.

Avoid

- glossy materials
- plastic gradients
- sharp reflections
- glassmorphism
- neon effects

Prefer

- paper
- linen
- cotton
- wood
- earth
- stone
- natural fibers

The interface should appear touched by time.

---

# Color Behavior

Color is descriptive,
not decorative.

Every color should imply material.

Example

Earth
→ soil
→ bark

Indigo
→ natural dye

Cotton
→ fiber

Background
→ paper

Avoid colors that exist only for aesthetic variation.

---

# Shape Language

Primary geometry

Rectangle

Secondary geometry

Organic edges created by imagery,
never by CSS border-radius.

Rules

- avoid pills
- avoid floating circles
- avoid playful blobs
- avoid excessive rounded corners

The layout should feel constructed.

Nature introduces irregularity through imagery,
not UI chrome.

---

# Border Language

Borders are editorial tools.

Allowed

- hairline borders
- inset borders
- quiet framing

Avoid

- thick outlines
- card borders
- colorful strokes

Borders define relationships,
not emphasis.

---

# Shadow Language

Shadows should suggest depth,
not elevation.

Allowed

- extremely soft shadows
- almost imperceptible

Avoid

- floating cards
- dramatic drop shadows
- material-design elevation

Depth comes from overlap,
not shadow.

---

# Typography Relationship

Typography should never dominate imagery.

Text exists to guide observation.

The user should

look

↓

observe

↓

read

never

read

↓

look

---

# Image Cropping

Images should feel discovered.

Avoid

- perfectly centered crops
- predictable framing

Prefer

- partial crops
- macro details
- edge bleeding
- implied continuation outside viewport

The world should always feel larger than the screen.

---

# Negative Space

Negative space represents atmosphere.

It is not empty.

It carries

- silence
- air
- time
- anticipation

Do not fill space merely because it exists.

---

# Layer Interaction

Atmosphere supports Subject.

Subject receives Annotation.

Structure quietly aligns all three.

No layer competes with another.

Visual attention should naturally descend through these layers.

---

# Human Review Checklist

Before approving any section, ask:

- Does this feel like an editorial publication instead of an app?
- Does every surface imply a real material?
- Is texture supporting the story rather than decorating it?
- Does the page remain calm despite visual richness?
- Could any border, shadow, or effect be removed without harming the narrative?
- Does every annotation feel attached to its subject?
- Does the atmosphere remain present even when almost invisible?

If the answer to every question is "yes",
the surface language is being respected.
